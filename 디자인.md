# UI/UX 개념 정리

## UI (User Interface)
- 사용자가 디지털 제품(앱, 웹사이트 등)을 사용할 때 **직접적으로 상호작용하는 화면 요소**
- 예: 버튼, 메뉴, 폰트, 색상 등

## UX (User Experience)
- 사용자가 제품이나 서비스를 **이용하며 겪는 모든 경험**
- 사용성, 편의성, 감정 등 종합적인 만족도 포함

## UI와 UX의 차이점 및 관계
| UI | UX |
|----|----|
| 시각적/기능적 인터페이스에 집중 | 사용자 경험 전반에 집중 |
| 제품의 외형을 다룸 | 제품 사용의 흐름과 느낌을 다룸 |
| UX의 한 부분 | UI를 포함한 더 넓은 개념 |

# 사용자 경험이 좋은 디자인의 특징

## 1. 직관적인 인터페이스
- **설명 없이도** 사용자가 기능을 이해하고 사용할 수 있는 구조
- 아이콘과 텍스트의 조화, 자연스러운 사용자 흐름

## 2. 일관된 레이아웃
- 화면 전환 시 구성 요소의 **위치와 스타일** 유지
- 예: 고정된 네비게이션 바, 동일한 버튼 스타일

## 3. 명확한 피드백 제공
- 사용자 행동에 대한 즉각적이고 **명확한** 응답 제공
- 예: 로딩 표시, 오류 메시지

## 4. 접근성 고려
- 다양한 사용자의 상황(시각, 청각 등)을 고려한 디자인
- 예: 대비 높은 색상

# UX Writing 원칙

## UX Writing이란?
디지털 제품에서 사용자와 소통하는 모든 문구 작성
## 핵심 원칙

### 1. 명확성
- 쉽게 이해 가능한 언어 사용
- 예: "계속 진행하기"보다 "시작하기"

### 2. 간결성
- 불필요한 단어 제거, **짧고 효율적**인 문장
- 예: "비밀번호를 입력하세요" → "비밀번호 입력"

### 3. 일관성
- 동일한 상황에는 항상 **동일한** 표현 사용
- 예: "로그인"과 "접속"을 혼용하지 않기

### 4. 사용자 이해
- 사용자 관점에서 따뜻하고 배려 있는 문장 구성
- 예: "입력이 잘못되었습니다" → "비밀번호가 틀렸어요. 다시 확인해 주세요."

### 5. 적절한 피드백 제공
- 사용자의 행동 결과에 대한 정확하고 도움 되는 문구 작성

# Figma 유용한 정보 정리

## 오토레이아웃 (Auto Layout)
- 요소 간 간격 자동 정렬 및 크기 조정
- 버튼이나 카드 등 반응형 디자인에 유용
- 단축키: `Shift + A`

### 주요 설정
- 방향: 수직 / 수평
- Spacing: 요소 사이 간격 조절
- Padding: 내부 여백 설정

## 컴포넌트 (Component)
- 반복되는 UI 요소를 재사용 가능한 단위로 관리
- 마스터 컴포넌트 수정 시 모든 인스턴스에 반영
- 단축키: `Ctrl + Alt + K`

### 활용 팁
- 버튼, 아이콘 세트, 카드 등을 컴포넌트로 생성
- `Ctrl + Enter`: 인스턴스 내부 편집

## 자주 쓰는 단축키

| 기능 | Windows |
|------|---------|
| 오토레이아웃 적용 | Shift + A |
| 컴포넌트 만들기 | Ctrl + Alt + K |
| 그룹 묶기 | Ctrl + G |
| 가운데 정렬 등 정렬 기능 | Shift + A → 옵션 선택 |
| 눈금자 보기 | Ctrl + R |
| 가이드라인 표시 | Shift + R |

## 그리드 (Grid)

콘텐츠를 정렬하고 배치하기 위한 보이지 않는 기준선 또는 프레임

**목적**  
- 일관된 레이아웃 유지  
- 콘텐츠 가독성 향상  
- 반응형 디자인 구현 용이

**종류**  
- 컬럼 그리드  
- 모듈 그리드  
- 계층 그리드

## 여백 (Whitespace)
 
텍스트, 이미지, 요소 사이의 빈 공간

**역할**  
- 콘텐츠 집중 유도  
- 시각적 안정성 제공  
- 정보 간의 구분 용이

## 정렬 (Alignment)

요소를 특정 기준선에 따라 배치하는 방식

**종류**  
- 좌측 정렬  
- 우측 정렬  
- 중앙 정렬  
- 양쪽 정렬

**효과**  
- 콘텐츠 간의 시각적 질서 유지  
- 사용자 흐름 유도

## 대비 (Contrast)

서로 다른 시각적 특성 간의 차이로 인한 구분 효과

**종류**  
- 색상 대비  
- 크기 대비  
- 굵기 대비  
- 형태 대비

**목적**  
- 중요한 요소 강조  
- 정보 구조 명확화

## 반복 (Repetition)

동일한 디자인 요소를 여러 곳에 반복적으로 사용하는 기법

**목적**  
- 시각적 일관성 제공  
- 브랜드 정체성 강화

**예시**  
- 동일한 버튼 스타일 반복  
- 같은 색상 및 글꼴 유지

## 색의 대비 (Color Contrast)

서로 다른 색상 간의 시각적 차이

**종류**  
- 보색 대비: 색상환에서 서로 반대 위치  
- 명도 대비: 밝고 어두운 색 차이  
- 채도 대비: 선명함의 차이

**목적**  
- 텍스트 가독성 향상  
- 강조 효과

## 색의 조화 (Color Harmony)

서로 잘 어울리는 색상 조합을 구성하는 원리

**종류**  
- 보색 조화  
- 유사색 조화  
- 삼각형 조화  
- 단색 조화

**효과**  
- 안정감 있는 디자인  
- 통일성 있는 색 구성

## 색의 심리적 효과

| 색상 | 효과 | 활용 분야 |
|------|------|-----------|
| 빨강 | 열정, 자극, 긴장 | 경고, 세일, 에너지 브랜드 |
| 파랑 | 안정감, 신뢰 | 금융, IT, 의료 |
| 초록 | 자연, 안정, 성장 | 환경, 식품, 건강 |
| 노랑 | 경쾌함, 경고 | 하이라이트, 주의 요소 |
| 보라 | 신비, 창의성 | 예술, 뷰티 |
| 검정 | 고급, 권위 | 패션, 럭셔리 |
| 흰색 | 순수, 단정 | 의료, 미니멀 디자인 |

## 감산혼합 (Subtractive Color Mixing)

**정의**  
색소가 빛을 흡수해 색을 만드는 방식

**기본 색상**  
- Cyan 
- Magenta  
- Yellow 

**특징**  
- 색을 섞을수록 어두워짐  
- CMY 모두 섞으면 검정(이론상)  
- 실제 인쇄에서는 K(검정)을 별도로 추가 → CMYK

**사용 예시**  
- 인쇄물, 프린터, 페인트

## 가산혼합 (Additive Color Mixing)

**정의**  
빛의 색을 더해 색을 표현하는 방식

**기본 색상**  
- Red (빨강)  
- Green (초록)  
- Blue (파랑)

**특징**  
- 색을 섞을수록 밝아짐  
- RGB 모두 섞으면 흰색

**사용 예시**  
- 모니터, TV, 스마트폰, LED 조명

## 색상 표현 방식의 이해

### RGB
- 용도: 디지털 디스플레이  
- 예시: `rgb(255, 0, 0)` 

### HEX
- 용도: 웹에서 널리 사용되는 색상 표현 방식  
- 예시: `#FF0000` 

### HSL
- 용도: 색상(H), 채도(S), 명도(L)를 기준으로 색을 정의  
- 예시: `hsl(0, 100%, 50%)` 

### CMYK 
- 용도: 인쇄용 색상 모델  
- 예시: `cmyk(0%, 100%, 100%, 0%)`
